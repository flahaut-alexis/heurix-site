Comment ça marche

Comment est comptée une requête ?

Simple par principe : une requête correspond à une action concrète du moteur.

RechercheChaque recherche tapée par un visiteur compte pour une requête (le moteur répond dès la 3e lettre par défaut, comme la plupart des moteurs modernes)
IndexationIndexer un lot de 1 000 produits compte pour 10 requêtes
Appels APIChaque appel à l'API de gestion ou de recherche compte pour une requête
DépassementAu-delà du volume inclus dans votre forfait, le surplus est facturé au tarif indiqué sur votre plan — jamais de coupure brutale du service

Questions fréquentes

Search API vs Professional Services.

Quelle est la différence avec les Professional Services ?

Le Search API, c'est le moteur en libre-service : vous l'intégrez vous-même, vous gérez la configuration depuis votre back-office. Les Professional Services (audit, intégration, formation — 850 € HT/jour), c'est un accompagnement humain : quelqu'un configure avec vous, sur votre solution actuelle ou sur le moteur Heurix. Les deux sont complémentaires — beaucoup de clients commencent par une mission d'audit, puis passent au Search API pour la suite.

Puis-je changer de forfait ou arrêter à tout moment ?

Oui. Facturation mensuelle, aucun engagement de durée. Vous changez de forfait ou vous arrêtez quand vous voulez, sans pénalité.

Le moteur propriétaire remplace-t-il Algolia ou Elasticsearch ?

Pas nécessairement. Si votre solution actuelle est bien configurée, il n'y a aucune raison d'en changer. Le Search API a du sens quand vous n'avez pas encore de moteur dédié, ou quand vous cherchez une alternative plus simple à opérer pour un catalogue de taille PME/mid-market.

Que se passe-t-il si je dépasse mon quota de requêtes ?

Le service continue de fonctionner normalement — le surplus est simplement facturé au tarif par tranche indiqué sur votre forfait. Vous êtes prévenu avant tout dépassement significatif, pas de coupure surprise.

Puis-je passer du Search API à un accompagnement complet ?

Oui, à tout moment. Si la configuration en self-service atteint ses limites (catalogue très technique, besoin de règles de merchandising avancées), on peut prendre le relais avec une mission ciblée, sur la base de ce qui est déjà en place.
