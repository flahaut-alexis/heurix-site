Connecter Heurix à Claude Desktop et Cursor avec le serveur MCP

Posez une question en langage naturel sur votre catalogue directement dans Claude Desktop ou Cursor — pas d'appel API à écrire, l'agent fait le pont. Quinze minutes d'installation, testé de bout en bout avant publication.

Ce qu'il vous faut

Python 3.10 ou plus sur la machine où tourne votre agent (votre ordinateur, pas un serveur — le serveur MCP se lance localement, à la demande), et une clé API Heurix.

Étape 1 — Téléchargez et installez

Téléchargez le serveur MCP, dézippez-le à un endroit stable (pas dans un dossier de téléchargements temporaire), puis installez les dépendances :

cd heurix-mcp-server
pip3 install -r requirements.txt
which python3

Gardez le résultat de which python3 sous le coude — vous en aurez besoin à l'étape suivante, en chemin complet.

Étape 2 — Configurez Claude Desktop

Réglages → Développeur → Modifier la configuration (crée le fichier s'il n'existe pas). Sur macOS, il se trouve à ~/Library/Application Support/Claude/claude_desktop_config.json ; sur Windows, %APPDATA%\Claude\claude_desktop_config.json.

{
  "mcpServers": {
    "heurix": {
      "command": "/usr/bin/python3",
      "args": ["/chemin/complet/vers/heurix-mcp-server/server.py"],
      "env": {
        "HEURIX_API_KEY": "hx_votre_cle_api",
        "HEURIX_API_BASE": "https://api.heurix.fr"
      }
    }
  }
}

Le chemin complet vers python3 n'est pas optionnel — Claude Desktop lance la configuration avec un PATH minimal, une commande courte qui fonctionne dans votre terminal peut échouer silencieusement ici. Même chose pour le chemin vers server.py : complet, jamais relatif (pas de ~).

Redémarrez Claude Desktop entièrement (Cmd/Ctrl+Q, pas juste fermer la fenêtre).

Et dans Cursor ?

Même structure de configuration, dans .cursor/mcp.json (à la racine d'un projet) ou ~/.cursor/mcp.json (global, tous projets). Ensuite : Réglages Cursor → Tools & MCP, vérifiez que "Enable MCP Servers" est coché et que heurix apparaît avec un statut connecté. En cas de souci, le panneau Output → MCP affiche les logs bruts du serveur — souvent plus parlant que l'interface elle-même.

Étape 3 — Testez

Dans une nouvelle conversation, sans rien préciser de plus qu'un nom de catalogue :

J'ai un catalogue Heurix qui s'appelle moncatalogue.
Est-ce que j'ai des vis M8 en stock, et à quel prix ?

Derrière l'échange, l'agent appelle heurix_search(catalog="moncatalogue", query="vis M8"), qui relaie vers votre API Heurix exactement comme le ferait n'importe quel appel curl — la réponse revient ensuite reformulée en langage naturel, avec vos vraies références et vos vrais prix.

Les trois outils disponibles

heurix_search (recherche par mot-clé, tolérante aux fautes de frappe), heurix_browse (produits d'une catégorie, avec tri — stock, prix, popularité...), et heurix_catalog_stats (liste les catalogues et catégories disponibles — utile en premier appel si l'agent ne connaît pas encore votre structure). Chaque appel de recherche ou de Browse consomme le quota normal de la clé utilisée, comme un appel direct à l'API.

Sécurité

Votre clé API vit uniquement dans le fichier de configuration local du client MCP, jamais transmise en clair dans un appel d'outil — le serveur la lit une fois au démarrage et l'utilise pour chaque appel vers l'API Heurix. Documentation complète des trois outils sur la référence API.
