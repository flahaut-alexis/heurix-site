Indexez votre premier catalogue outillage en 5 minutes

Deux appels API, un vrai résultat de recherche avec faute de frappe et format différent. Tout le code de cette page est testé — copiez-collez, adaptez à votre catalogue.

Ce qu'il vous faut

Une clé API (essai gratuit 14 jours, sans carte bancaire) et un terminal. C'est tout — pas de SDK à installer, le moteur répond en JSON brut sur des appels HTTP standards.

Étape 1 — Indexez 4 produits

Un petit catalogue de quincaillerie, avec le pack de règles outillage (celui qui comprend les diamètres, longueurs et matières dans une référence) :

curl -X POST https://api.heurix.fr/v1/index/quincaillerie/items \
  -H "Authorization: Bearer VOTRE_CLE_API" \
  -H "Content-Type: application/json" \
  -d '{
  "rulepack": "outillage",
  "items": [
    {"id": "V001", "name": "Vis à métaux tête hexagonale", "ref": "M8 x 20 - Inox A2", "stock": 120},
    {"id": "V002", "name": "Vis à métaux tête hexagonale", "ref": "M8 x 30 - Inox A2", "stock": 45},
    {"id": "R001", "name": "Roulement à billes étanche", "ref": "6205-2RS", "stock": 8},
    {"id": "E001", "name": "Écrou hexagonal", "ref": "M8 - Zingué", "stock": 300}
  ]
}'

Réponse :

{"indexed": 4, "catalog": {"catalog": "quincaillerie", "products": 4,
 "terms": 18, "annotations": 11, "rulepack": "outillage", "synonym_groups": 0}}

11 annotations générées automatiquement à partir de 4 produits — c'est la cascade de règles qui vient de décomposer chaque référence (diamètre, longueur, matière) sans que vous ayez écrit la moindre regex.

Étape 2 — Cherchez, avec une faute et un format différent

Le vrai test : pas la référence exacte telle qu'indexée, mais ce qu'un client tape réellement — une faute sur « vis », et « M8x20 » collé sans espaces ni tirets :

curl -X POST https://api.heurix.fr/v1/index/quincaillerie/search \
  -H "Authorization: Bearer VOTRE_CLE_API" \
  -H "Content-Type: application/json" \
  -d '{"q": "vs m8x20 inox"}'

Réponse (raccourcie) :

{
  "total": 3,
  "hits": [
    {
      "product": {"id": "V001", "ref": "M8 x 20 - Inox A2", "stock": 120},
      "score": 43.5,
      "matched": [
        "terme 'inox' (depuis 'inox', x1.00)",
        "annotation #LONG_20",
        "annotation #DIAM_M8",
        "annotation #VIS_M8X20",
        "annotation #DIAM_M8_INOX",
        "annotation #MAT_INOX"
      ]
    },
    { "product": {"id": "V002", "ref": "M8 x 30 - Inox A2"}, "score": 28.5, ... },
    { "product": {"id": "E001", "ref": "M8 - Zingué"}, "score": 7.5, ... }
  ]
}

Ce qui vient de se passer

« vs m8x20 inox » n'existe littéralement nulle part dans les données indexées — ni « vs » (faute sur « vis »), ni « m8x20 » sous cette forme exacte (la référence stockée est « M8 x 20 »). Le moteur a quand même trouvé et classé en premier le bon produit, et le champ matched dit précisément pourquoi : il a reconnu un diamètre (DIAM_M8), une longueur (LONG_20), leur combinaison en tant que référence de vis (VIS_M8X20), et la matière (MAT_INOX) — quatre annotations qui pèsent bien plus dans le score que la simple correction orthographique de « vis ». C'est la différence entre un moteur qui tolère les fautes sur des mots, et un moteur qui comprend la structure d'une référence.

Et avec votre vrai catalogue ?

Remplacez le contenu du tableau items par vos propres produits (5 000 par appel maximum, plusieurs appels pour un catalogue plus grand), gardez le pack outillage s'il correspond à votre secteur — ou mode, industrie pour les deux autres packs fournis. La documentation complète couvre les facettes, les synonymes métier et la recherche fédérée sur plusieurs catalogues.
